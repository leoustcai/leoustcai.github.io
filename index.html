Projects

The active slate.

These are not moodboard concepts. They are the actual repositories in the workspace, grouped by what they are trying to improve right now.

Tooling

git2

A lighter wrapper around the system Git binary that keeps one custom workflow: `branch2`. The point is simple value with much lower maintenance cost.

Why it matters

It trims the ambition down to the useful core and keeps daily Git usage familiar.

Python CLI branch mirroring dev-friendly packaging
Tooling

git2-upstream-sync

The upstream-facing fork that stays close to Git while exploring a shared-filesystem workflow for multi-agent development.

Why it matters

It protects compatibility while creating room for faster collaboration patterns.

upstream sync build automation Git-compatible fork
Platform

openclaw

A personal assistant platform that runs on your own devices and meets people on the channels they already use, from chat apps to voice and live canvas surfaces.

Why it matters

It turns “assistant” from a demo concept into an everyday product with real presence.

multi-channel self-hosted feel always-on assistant
Library

skills-collection

A living library of reusable skills and operating notes, built to keep the best workflows portable instead of trapped in one project.

Why it matters

It captures repeatable work so the team gets better without starting from zero each time.

reusable skills workflow notes shared patterns
Web

mywebsite

The front door. A simple static page that should introduce the work cleanly, sound human, and make the team feel like people with a point of view.

Why it matters
